Formal Islands
Certified local cores inside natural-language mathematical proofs

What Is This?

Most mathematical proofs are written informally — definitions are left implicit, steps rely on shared notation, and the overall argument is human-readable but not machine-checkable. Full formalization in a proof assistant like Lean is powerful but expensive: it often requires months of infrastructure work just to state a theorem in a way the system accepts.

Formal Islands targets the middle ground. Given a theorem statement and an informal proof, the pipeline plans a small proof graph, identifies which steps are concrete enough to certify today, and runs a Lean verification attempt on each one. The result is a mixed artifact: parts of the proof are formally verified; the rest is clearly labelled as informal and given an honest review checklist.

A formal island is a node in the proof graph that has been independently verified in Lean — certified in isolation, without requiring the rest of the proof to be formalized first.

This is especially useful for analysis, PDEs, and variational arguments where the global proof is too large to formalize end-to-end, but local steps are concrete enough to verify now.

Quick Start

Install the package and its dependencies:

git clone https://github.com/aditya-ramabadran/formal-islands.git
cd formal-islands
python3 -m venv .venv
.venv/bin/python -m pip install -e '.[dev]'

You'll also need a local Lean/Mathlib workspace (see the full setup guide) and at least one backend — Claude Code, Gemini, or Codex for planning; Aristotle for formalization.

Run the pipeline interactively on your own theorem:

formal-islands new --backends claude/aristotle

Or run against a featured example:

formal-islands run two_point_log_sobolev --backends claude/aristotle --max-attempts 4

The output directory, workspace path, and most options are inferred automatically. See the README for the full flag reference.
